## מהן מיומנויות?

**מיומנות** היא חבילה מקיפה וניידת של הוראות שנותנת לסוכן AI מומחיות בתחום או משימה מסוימים. מיומנויות הן הבלוקים הניתנים לשימוש חוזר של סוכנים: בונים אותן פעם אחת, וכל סוכן יכול להשתמש בהן.

### 2. מבנה ידע בצורה היררכית

ארגן מידע מהכללי לספציפי:

```markdown
# API Design Skill

## Core Principles
- Resources should be nouns, not verbs
- Use HTTP methods semantically
- Version your APIs from day one

## Detailed Guidelines
[More specific rules...]

## Reference Materials
- See `rest-conventions.md` for naming conventions
- See `error-codes.md` for standard error responses
```

### 3. כלול דוגמאות קונקרטיות

כללים מופשטים הופכים ברורים עם דוגמאות:

```markdown
## Endpoint Naming

✅ Good:
- GET /users/{id}
- POST /orders
- DELETE /products/{id}/reviews/{reviewId}

❌ Avoid:
- GET /getUser
- POST /createNewOrder
- DELETE /removeProductReview
```

### 4. ספק מסגרות לקבלת החלטות

עזור לסוכן לבצע בחירות במצבים עמומים:

```markdown
## When to Use Pagination

Use pagination when:
- Collection could exceed 100 items
- Response size impacts performance
- Client may not need all items

Use full response when:
- Collection is always small (<20 items)
- Client typically needs everything
- Real-time consistency is critical
```

### 5. הוסף דפוסי התאוששות

צפה מראש מה יכול להשתבש:

```markdown
## Common Issues

**Problem**: Client needs fields not in standard response
**Solution**: Implement field selection: GET /users?fields=id,name,email

**Problem**: Breaking changes needed
**Solution**: Create new version, deprecate old with timeline
```

- **לשפר** מיומנויות בהתבסס על שימוש בעולם האמיתי

## שיטות עבודה מומלצות

### לבניית מיומנויות

1. **התחל ספציפי, ואז הכלל** - בנה מיומנות למקרה השימוש המדויק שלך תחילה, ואז הפשט
2. **כלול מקרי כישלון** - תעד מה המיומנות לא יכולה לעשות ואיך להתמודד עם זה
3. **נהל גרסאות של המיומנויות שלך** - עקוב אחר שינויים כדי שסוכנים יוכלו להסתמך על גרסאות יציבות
4. **בדוק עם משימות אמיתיות** - אמת מיומנויות מול עבודה בפועל, לא רק תיאוריה

### לשימוש במיומנויות עם סוכנים

1. **קרא את המיומנות תחילה** - הבן מה מיומנות עושה לפני שפורסים אותה
2. **התאם אישית בתבונה** - דרוס ברירות מחדל של מיומנות רק כאשר הכרחי
3. **עקוב אחר ביצועים** - עקוב אחר כמה טוב מיומנויות מתפקדות בהקשר שלך
4. **תרום שיפורים** - כאשר אתה משפר מיומנות, שקול לשתף בחזרה
